  <action>Run `shellcheck` against every `.sh` file Phase 1 touched:

```bash
shellcheck install.sh uninstall.sh hooks/session-start.sh hooks/post-compact.sh config/statusline.sh scripts/check-version-drift.sh tests/fixtures/hooks/setup-fixtures.sh
```

For each warning:
1. **Read the SC code documentation:** `https://www.shellcheck.net/wiki/SC<NNNN>` — understand what the warning means before silencing it.
2. **Prefer fixing over disabling.** Most warnings (SC2086 unquoted vars, SC2046 word-splitting in `$()`, SC2155 declare-and-assign, SC2034 unused vars) have a code fix that makes the script safer.
3. **If a warning is genuinely a false positive** (e.g., SC1091 sourced file unknown at parse, SC2034 documented-unused export), add an inline disable comment immediately above the offending line:
   ```bash
   # shellcheck disable=SC1091  # sourced at install time, may not exist on first run
   . "$CLAUDE_DIR/some-file"
   ```
   AND add a corresponding entry in `.shellcheckrc` ONLY if the disable is global (e.g., a project-wide convention) — most disables should be inline-scoped, not global.

**Anti-pattern: mass disable.** Do not bulk-add disables to `.shellcheckrc` without per-warning justification. If shellcheck flags 50 warnings, fix 49 and disable 1, not the reverse.

**Common Phase 1 warnings (anticipated):**
- **SC2086** (Double quote to prevent globbing) — fix: quote the variable. `cp $src $dst` → `cp "$src" "$dst"`.
- **SC2046** (Quote to prevent word splitting) — fix: quote the command substitution. `cd $(pwd)` → `cd "$(pwd)"`.
- **SC2155** (Declare and assign separately) — fix: split the line. `local x=$(cmd)` → `local x; x=$(cmd)`.
- **SC2034** (var appears unused) — fix: use it, or `# shellcheck disable=SC2034` with rationale, or delete it.
- **SC1090/SC1091** (sourced file) — fix: `# shellcheck source=path/to/file` or `disable=SC1091` if the file legitimately may not exist.
- **SC2162** (read without -r) — fix: add `-r` (we use `read -rp` already in prompt_overwrite — verify all read calls use `-r`).

Iterate until `shellcheck install.sh uninstall.sh hooks/*.sh config/statusline.sh scripts/*.sh tests/fixtures/hooks/*.sh; echo $?` prints `0`.

Each fix lands as its own atomic commit (one commit per file fixed, OR one commit per SC code if the same code is fixed across multiple files in one pass). Commit message format: `fix(shellcheck): <SC code> in <file or area> (FOUND-08)`.</action>
