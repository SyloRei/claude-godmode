# PRD: Consolidate All Plugin Artifacts into `.claude-pipeline/`

## 1. Introduction/Overview

All artifacts generated by the claude-godmode plugin (PRDs, stories, progress tracking, archives) should live under a single `.claude-pipeline/` directory in consumer projects. This directory should be automatically added to `.gitignore` so plugin-generated files never pollute the consumer's version control.

Currently, PRDs are written to `tasks/` while pipeline artifacts (`stories.json`, `progress.txt`, archives) already use `.claude-pipeline/`. This change consolidates everything under `.claude-pipeline/` and adds automatic `.gitignore` management.

## 2. Goals

- All plugin-generated artifacts write to `.claude-pipeline/` and nowhere else
- PRDs move from `tasks/prd-*.md` to `.claude-pipeline/prds/prd-*.md`
- The `tasks/` directory concept is fully removed from the plugin
- `.claude-pipeline/` is automatically added to consumer project `.gitignore` when the pipeline is first used
- All documentation reflects the new unified path structure

## 3. User Stories

### US-001: PRDs Write to `.claude-pipeline/prds/`

**Description:** As a plugin user, I want PRDs saved under `.claude-pipeline/prds/` so that all plugin artifacts are in one place.

**Acceptance Criteria:**
- [ ] `/prd` skill saves PRDs to `.claude-pipeline/prds/prd-[feature-name].md`
- [ ] `/prd` skill creates `.claude-pipeline/prds/` directory if it doesn't exist
- [ ] All references to `tasks/` in `skills/prd/SKILL.md` are replaced with `.claude-pipeline/prds/`
- [ ] All quality gates pass (typecheck, lint, tests)

### US-002: Auto-Add `.claude-pipeline/` to `.gitignore` on First Pipeline Use

**Description:** As a plugin user, I want `.claude-pipeline/` automatically added to my project's `.gitignore` when I first use the pipeline, so I don't have to remember to do it myself.

**Acceptance Criteria:**
- [ ] `.gitignore` management is triggered from `/prd` and `/plan-stories` skills (the entry points that create `.claude-pipeline/`), NOT from session-start hook
- [ ] Check runs only when inside a git repository
- [ ] Uses exact-line matching (`grep -qxF '.claude-pipeline/'`) to avoid false positives from comments, partial strings, or similar entries
- [ ] Handles missing trailing newline in existing `.gitignore` (use `printf` with leading newline guard)
- [ ] If `.gitignore` exists but doesn't contain the entry, append with a comment header (`# claude-godmode pipeline artifacts`)
- [ ] If `.gitignore` doesn't exist, create it with the entry
- [ ] Supports opt-out: skip if `.gitignore` contains `# claude-godmode: unmanaged`
- [ ] Idempotent — running multiple times doesn't duplicate the entry
- [ ] All quality gates pass (typecheck, lint, tests)

### US-003: Update `/plan-stories` Skill References and Add `prdSource` Field

**Description:** As a plugin maintainer, I want `/plan-stories` to reference the correct PRD location and track which PRD generated each stories.json.

**Acceptance Criteria:**
- [ ] `skills/plan-stories/SKILL.md` reads PRDs from `.claude-pipeline/prds/` instead of `tasks/`
- [ ] Any instructions that tell the agent where to find PRDs point to `.claude-pipeline/prds/`
- [ ] Stories output path remains `.claude-pipeline/stories.json` (no change needed)
- [ ] `/plan-stories` records a `prdSource` field in `stories.json` pointing to the PRD file that was consumed (e.g., `"prdSource": ".claude-pipeline/prds/prd-user-auth.md"`)
- [ ] All quality gates pass (typecheck, lint, tests)

### US-004: Update All Documentation and Clean Up Plugin Repo

**Description:** As a plugin user or contributor, I want all docs to reflect the new artifact structure so there's no confusion about where files go.

**Acceptance Criteria:**
- [ ] `config/INSTRUCTIONS.md` — all `tasks/` references replaced with `.claude-pipeline/prds/`
- [ ] `README.md` — any references to artifact output paths updated
- [ ] `commands/godmode.md` — pipeline description reflects new paths
- [ ] `skills/execute/SKILL.md` — verify all paths are correct (should already reference `.claude-pipeline/`)
- [ ] `agents/executor.md` — verify paths are correct
- [ ] No remaining references to `tasks/` as an artifact output directory anywhere in the repo
- [ ] Delete `tasks/` directory from the plugin repo itself (move this PRD to `.claude-pipeline/prds/` first)
- [ ] All quality gates pass (typecheck, lint, tests)

### US-005: Update Archive Structure with PRD Tracking

**Description:** As a plugin user, I want archived pipeline runs to include the source PRD so that the full context of a previous run is preserved.

**Acceptance Criteria:**
- [ ] `skills/plan-stories/SKILL.md` archive instructions copy only the source PRD (identified by the `prdSource` field in `stories.json`) into the archive directory
- [ ] Archive path remains `.claude-pipeline/archive/YYYY-MM-DD-feature-name/`
- [ ] Archive includes: `stories.json`, `progress.txt`, and the source PRD file
- [ ] All quality gates pass (typecheck, lint, tests)

## 4. Functional Requirements

- FR-1: The `/prd` skill MUST save output to `.claude-pipeline/prds/prd-[feature-name].md`
- FR-2: The `/prd` skill MUST create the `.claude-pipeline/prds/` directory if it does not exist
- FR-3: The `/prd` and `/plan-stories` skills MUST ensure `.claude-pipeline/` is in `.gitignore` when first creating the directory (inside a git repo)
- FR-4: The `.gitignore` check MUST use exact-line matching (`grep -qxF`) to avoid false positives
- FR-5: The `.gitignore` check MUST handle missing trailing newline in existing files
- FR-6: The `.gitignore` entry MUST include a comment identifying its origin (`# claude-godmode pipeline artifacts`)
- FR-7: The `.gitignore` check MUST skip modification if opt-out marker `# claude-godmode: unmanaged` is present
- FR-8: The `.gitignore` entry MUST be idempotent — no duplicates on repeated runs
- FR-9: The `/plan-stories` skill MUST look for PRDs in `.claude-pipeline/prds/`
- FR-10: The `/plan-stories` skill MUST record `prdSource` in `stories.json` pointing to the consumed PRD
- FR-11: The archive process MUST copy only the source PRD (via `prdSource`) when archiving a previous pipeline run
- FR-12: Zero references to `tasks/` as an artifact output directory MUST remain in the codebase after completion

## 5. Non-Goals (Out of Scope)

- Migrating existing `tasks/` directories in consumer projects (users can move files manually)
- Changing the `.claude-pipeline/` directory name (already established convention)
- Modifying how `stories.json` or `progress.txt` work internally (beyond adding `prdSource`)
- Adding a config option to customize the artifact directory path
- Changing any agent behavior beyond path references
- Modifying `session-start.sh` hook (`.gitignore` management is in skills, not hooks)

## 6. Technical Considerations

- **No build/typecheck/lint for this project** — it's a pure Markdown/shell plugin with no compiled code
- **Quality gates for this project:** Shell script syntax check (`bash -n`), no broken internal references
- **`.gitignore` management lives in skills, not hooks** — the session-start hook remains read-only to avoid surprising side effects and failure-mode risks (the hook uses `set -euo pipefail`, so a write failure would kill project context injection)
- **Idempotency pattern:** Use `grep -qxF '.claude-pipeline/' .gitignore` for exact-line matching. Handle missing trailing newline with: `[ -s .gitignore ] && [ "$(tail -c1 .gitignore)" != "" ] && printf '\n' >> .gitignore` before appending
- **Opt-out:** Check `grep -qF '# claude-godmode: unmanaged' .gitignore` before modifying — allows teams that want `.claude-pipeline/` tracked to opt out
- **File creation:** When creating `.claude-pipeline/prds/`, use `mkdir -p` for safety
- **`prdSource` field:** Added to `stories.json` root object by `/plan-stories`, used by archive logic to copy only the relevant PRD
- **Backwards compatibility:** Not a concern — `tasks/` was never documented as a stable API, and the plugin is pre-1.0 adoption
- **Story dependencies:** US-001 through US-005 are not independently shippable — they should ship as a single PR

### Artifact Directory Structure (Final State)

```
.claude-pipeline/
  prds/
    prd-feature-name.md
  stories.json          # includes "prdSource": ".claude-pipeline/prds/prd-feature-name.md"
  progress.txt
  archive/
    YYYY-MM-DD-feature-name/
      stories.json
      progress.txt
      prd-feature-name.md   # source PRD only (from prdSource field)
```

## 7. Success Metrics

- `grep -r "tasks/" skills/ agents/ config/ commands/ hooks/` returns zero results related to artifact output
- `.gitignore` in a fresh consumer project gets `.claude-pipeline/` added on first `/prd` or `/plan-stories` use
- `.gitignore` is NOT modified if opt-out marker is present
- Full pipeline (`/prd` -> `/plan-stories` -> `/execute`) works end-to-end with new paths
- `stories.json` contains `prdSource` field after `/plan-stories` runs
